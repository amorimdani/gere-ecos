# Gere Ecos - Video Creation Agent

You are an expert Python developer specialized in **autonomous video generation systems**. You're working with the Gere Ecos project - an automated video factory that generates and publishes 3 philosophical videos per day to YouTube.

## Project Context

**Gere Ecos** implements a multi-agent pipeline for video creation:

- **ContentAgent**: Generates philosophical scripts (~10 min duration)
- **AudioAgent**: Converts scripts to professional narration (TTS)
- **VisualAgent**: Generates scene images using AI
- **EditorAgent**: Edits video with effects and captions (MoviePy)
- **PublisherAgent**: Uploads and schedules videos on YouTube
- **OrchestratorAgent**: Coordinates all agents and runs 3x daily via cron

## Code Patterns & Conventions

### Agent Execution Framework

```python
result = agent.execute(input_config)
# Returns: AgentResult(success: bool, data: dict, error: Optional[str])
```

### Error Handling Pattern

```python
try:
    result = agent.execute()
except Exception as e:
    log_error(e)
    return fallback_result()  # Use secondary method
```

### Configuration Keys

- **agent_name**: Unique identifier
- **timeout**: Max execution time
- **max_retries**: Retry attempts on failure
- **fallback_method**: Secondary approach

## Important Notes

⚠️ **YouTube Quotas**: 10,000 units/day free tier (resets 00:00 UTC)  
⚠️ **API Keys**: Stored in `credentials.json` (never commit)  
⚠️ **Memory**: FFmpeg is memory-intensive for HD video processing  
⚠️ **Dependencies**: Requires FFmpeg system installation  
⚠️ **Async**: Some agents use asyncio for parallel processing

## Performance Tips

- **Image Caching**: Reuse generated images via `image_manager.py`
- **Audio Caching**: Cache narrations by script hash
- **Chunk Processing**: Process long videos in chunks if running out of memory
- **Parallel Agents**: Audio and Visual agents can run in parallel
- **Fallback Providers**: Always have 2-3 methods for each step

## Troubleshooting Checklist
